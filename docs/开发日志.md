## 2026-07-13 — Phase 1: GEO 优化 + Schema 结构化数据（首发上线）

### 完成内容
- **Schema.org 结构化数据**：WebSite（含搜索框）、Organization、Person（作者实体：南方医科大学博士/广东省人民医院眼科）、Article（含 speakable）、BreadcrumbList 共 5 个 JSON-LD
- **robots.txt**：新增 GPTBot、ClaudeBot、DeepSeekBot、KimiBot、Bytespider、YisouSpider、PerplexityBot、bingbot、Googlebot AI 爬虫白名单
- **安全响应头**：X-Content-Type-Options: nosniff，X-Frame-Options: SAMEORIGIN
- **图片懒加载**：postThumb() 函数增加 `loading="lazy"`
- **IndexNow 推送**：生成 Key 文件，发文钩子（themeInit 内注册）
- **sitemap.xml**：独立 PHP 生成器，含 image:image、lastmod、priority
- **feed.json**：JSON Feed 1.1 标准输出
- **宝塔定时任务**：cron 每天 03:00 生成 sitemap + feed

### 遇到并修复的错误
1. HTTP 500：Typecho_Plugin 钩子在 functions.php 全局域注册导致 fatal error → 移到 themeInit() 内
2. 443 端口不监听：服务器存在两个 nginx（系统版 + 宝塔版）→ 使用宝塔版 `/www/server/nginx/sbin/nginx`
3. feed.json 返回 404：Typecho rewrite 规则拦截 → 排除 feed/sitemap/indexnow 路径
4. header.php heredoc 损坏：SSH 部署时 shell 转义导致 → 从 bak 恢复后用 sed 插入 Schema
5. BlogPosting headline/url 为 null：postSchema() 调用上下文不对 → 删除 post.php 中的调用

### 部署方式
- SSH heredoc 直传（后续改为 Git + deploy.sh）
